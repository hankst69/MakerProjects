Build wasm on windows for QT

July 28, 2020July 30, 2020 ~ Ronak SHAH

The motivation for me to write this post was limited help got for building C++ files using QT classes to get wasm i.e. web-assembly file. QT supports web-assembly using emscripten tool chain. QT documentation mentions that building QT sources and project on windows can be done using windows sub-system for linux (WSL) and so did many other existing help that I could find. I initially started build process on my windows machine using Ubuntu & Debian by enabling WSL(1) i.e. prior to Windows May 2020 update. The process was smooth with only one step to be done after installing your favorite linux distro was to install essential build tools:

sudo apt-get install python nodejs build-essential '^libxcb.*-
dev' libx11-xcb-dev libglu1-mesa-dev libxrender-dev libxi-dev

After this, all I need had to do was to follow the instructions on the QT for WebAssembly documentation. All QT sources got build properly. Now, I used an existing QT project which I build from Ubuntu. Here, the WSL did a great job by auto mounting all your hard-disk partitions on linux root folder at /mnt/ location.

But the limitations of using WSL method to build and generate the WASM file are:

  1. While working with other developers, all developers need to setup WSL, Linux distro and Linux build tools on their windows machine. This is definitely motivating.
  2. Whenever I updated the code, I had to switch to the Linux system get the updated wasm.
  3. Difficulty in setting up Jenkins pipeline to build and deploy the wasm on windows server once I update the code.
  4. Also, WSL does have limitation getting the CPU resources and so build takes more time compared to using the native OS.

So, I wanted to switch to native windows development environment. Initial several attempts gave me various weird issues while building QT sources and then the C++ code. After some initial failures, I felt as if the setup is getting little complex by fixing each new error that I was getting. This was also an indication that I was doing something different than the process mentioned in the documentation.

I started by updating the windows to May 2020 i.e. version 2004. I removed all my previous setups and created a clean state to start from scratch again. After reading the documentation very clearly, I started with small baby steps. Finally, I successfully got QT sources build and even generated wasm and js files for the QT project. Following is the steps that I took and also request you to follow it in the given sequence and one-step at a time.

Steps:

  • Emscripten tool chain. Install the latest tools. 
git clone https://github.com/emscripten-core/emsdk.git
cd emsdk
emsdk install latest
emsdk activate latest
emsdk_env.bat
  • Download following build related tools:
    • MinGW: https://osdn.net/projects/mingw/releases/
    • sed: http://gnuwin32.sourceforge.net/packages/sed.htm
    • Following are my installation locations:
MINGW : %USERPROFILE%\mingw64\bin
sed: C:\Program Files (x86)\GnuWin32\bin
emsdk: D:\src\emsdk
  • Now, add these paths to the %PATH% environment variable and then re-open a fresh command prompt. You should be able to run following:
em++ --version
mingw32-make
  • Install and configure the QT sources with the following commands. As we need to consume emsdk tool chain, we need to first set the proper environment before running the configure.
D:\src\emsdk\emsdk_env.bat
D:
cd src\
git clone git://code.qt.io/qt/qt5.git
cd qt5
configure -no-warnings-are-errors -xplatform wasm-emscripten -platform win32-g++ -nomake examples -prefix %CD%\qtbase
mingw32-make module-qtbase module-qtdeclarative

Now, you should be having the binaries build for QT wasm. Now, we will build a test QT project to generate the wasm, js and html files. Before running the qmake command, the command prompt should be set with the correct emsdk environment. qmake is responsible for generating the makeFile for MinGW.

D:\src\emsdk\emsdk_env.bat
cd D:\src\qt_creator\web_assembly_test
D:\src\qt5\qtbase\bin\qmake web_assembly_test.pro
mingw32-make

Voilà ! you just build the wasm, html and js files.
