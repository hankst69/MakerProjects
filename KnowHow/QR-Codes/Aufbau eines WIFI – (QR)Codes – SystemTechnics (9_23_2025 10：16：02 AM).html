Aufbau eines WIFI – (QR)Codes

Aufbau eines WIFI – (QR)Codes
23. März 202327. November 2023

Hier mal ein kleiner Merker für mich wie man einen WIFI (Verbindung- QR)Code sich erstellen kann.
Der Code zum verbinden eines WLAN ist wie folgt aufgebaut:

    1. WIFI: Service Tag
    2. T: Transmission (WPA, WEP, nopass)
    3. S: SSID (Leerzeichen sind möglich!)
    4. P: Password
    5. H: Hidden/visible – true/false

Auf der Konsole lässt sich z.B. mit qrencode (brew install qrencode) ganz einfach einen QR Code erstellen.

echo "WIFI:T:WPA;S:Indernet;P:SuperGeheimesPasswort;;" | qrencode -t ansiutf8
echo "WIFI:T:WPA;S:Indernet;P:SuperGeheimesPasswort;;" | qrencode -t ansiutf8
echo "WIFI:T:WPA;S:Indernet;P:SuperGeheimesPasswort;;" | qrencode -t ansiutf8

In dem Beispiel ist es 

  1. WIFI
  2. T = WPA
  3. SSID = Indernet
  4. Das Passwort für das WLAN
  5. Ist sichtbar (muss nichts eingetragen werden)

Wer mag kann es natürlich auch als PNG-Bilddatei gleich abspeichern.

echo "WIFI:T:WPA;S:Indernet;P:SuperGeheimesPasswort;;" | qrencode -t png -o wifi-indernet.png
echo "WIFI:T:WPA;S:Indernet;P:SuperGeheimesPasswort;;" | qrencode -t png -o wifi-indernet.png
echo "WIFI:T:WPA;S:Indernet;P:SuperGeheimesPasswort;;" | qrencode -t png -o wifi-indernet.png

Anschließend lässt sich der QR Code mit dem Mobil-Telefon scannen und das WLAN sich verbinden.
